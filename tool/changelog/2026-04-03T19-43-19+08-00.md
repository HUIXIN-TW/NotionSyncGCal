Datetime: 2026-04-03T19:43:19+08:00

What changed:

- Added `src/utils/token_crypto.py` with encrypted-token compatibility helpers:
  - Detect `enc:v1` payloads.
  - Decrypt encrypted tokens when `TOKEN_ENCRYPTION_KEY` is configured.
  - Keep plaintext tokens as-is for backward compatibility.
  - Encrypt tokens on write when key is configured.
- Updated `src/utils/dynamodb_utils.py`:
  - Decrypt `accessToken`/`refreshToken` when reading Google and Notion OAuth token records.
  - Encrypt refreshed Google tokens before saving back to DynamoDB when key is present.
- Added `test/test_token_crypto.py` to validate encrypted/plaintext fallback behavior.
- Added `cryptography==44.0.3` to `requirements.txt` for AES-GCM support.

Why:

- Sync Lambda now needs to read mixed token formats during migration (legacy plaintext + new encrypted tokens).
- Prevent `invalid_grant` caused by passing encrypted token payloads directly to Google APIs.

Risk:

- Low to medium. Token read/write behavior now depends on `TOKEN_ENCRYPTION_KEY` when encrypted payloads are present.
- If encrypted data exists but key is missing/invalid, Lambda will fail fast with explicit token-crypto errors.
